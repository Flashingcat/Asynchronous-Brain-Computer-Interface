# 异步运动想象评估协议

## 1. 文档目的

本文档规定本项目评估代码必须遵守的顶层协议，用于避免数据泄漏、时间穿越和指标口径混用。

当前验证范围为 BNCI2014001 的 Subject 1–9。模型结构不是本评估工作的研究对象；原仓库的两阶段分类器作为首个基线。

本协议的所有正式结果均基于官方 trial 级伪迹标记清洗后生成的冻结数据；该前提不代表系统已经实现在线伪迹检测。

本协议区分两种正式结论和一种中间诊断，报告结果时不得混称：

1. 离线窗口级结果；
2. 因果窗口级诊断结果；
3. 严格在线仿真结果。

其中第 2 项使用在线模型的因果输入和 logit，但不运行连续决策器，用于分离“单窗分类能力”和“时间聚合、阈值及状态机”的影响，不作为独立系统结论。

## 2. 固定数据边界

- 数据集：BNCI2014001；
- 被试：Subject 1–9；
- 训练来源：`0train` session；
- 最终测试来源：`1test` session；
- 每个 MAT 文件按原始顺序选取恰好包含 48 个 trial 的 6 个记录，并依次映射为任务 run 0 至 5；无 trial 的眼动校准记录不得作为 `IDLE`。Subject 1 的任务记录对应 `data[3]` 至 `data[8]`，其他被试不得假定相同固定下标；
- 分类输入只使用前 22 个 EEG 通道，3 个 EOG 通道不得作为分类特征；
- 正式协议使用原生 250 Hz，不重采样；2 秒窗口包含 500 点，0.5 秒步长包含 125 点；
- 兼容配置允许沿用 MNE 重采样至 128 Hz，得到 256 点窗口和 64 点步长，但必须标为非严格因果的兼容结果；
- 最终测试必须将 `1test` 的 6 个 run 分别从头到尾回放；
- 每个 run 开始时重置因果滤波缓存、历史分数缓存和决策状态，禁止带入上一 run 的历史信息；
- run 之间保持模型参数和训练得到的标准化参数不变；
- 测试 session 不得参与模型选择、停止轮次选择、阈值选择或连续确认参数选择，以防止测试信息泄漏；
- 需要调节的参数只能由训练 session 内部的验证集确定；验证划分以完整任务 run 为最小单位，同一 run 的 trial、segment 和窗口不得跨集合，禁止随机按窗口或 trial 划分；
- 由数据估计的预处理参数——当前包括每个 EEG 通道的均值和标准差——只能由当前 fold 的训练 run 计算；
- 验证集和测试集只能应用训练得到的预处理参数，不得参与估计；当前实现把逐通道 mean/std 保存在 session0-only bundle 中，并由 checkpoint 的训练合同哈希绑定该 bundle。checkpoint 与对应 bundle 必须成套保存和使用，不得单独迁移后另配统计量。

当前历史数据构建必须如实披露：既有 `native250_artifact_trial_v1` 母索引曾同时打开并解析 `0train/1test`，因此本项目不能把 session 1 称为“历史上从未访问的盲测集”。已完成的训练器和策略评估器只消费 session0-only bundle，未把 session 1 的信号、标签或统计量用于拟合和选择；这里的“测试隔离”只表示不参与训练、调参和开发集成绩。正式 session 1 结果仍须在策略冻结后通过单独解锁流程生成，并称为 held-out session 评价。

### 2.1 训练 session 内部验证划分

每个被试 `0train` 的 6 个任务 run 分别采用固定的六折留一 run 验证，不使用随机划分：fold 编号等于验证 run 编号；当前 fold 以该 run 为验证集，其余 5 个 run 为训练集。每个 run 恰好作为验证集一次，因此该划分没有随机种子；不同被试不得混入同一被试的 fold。

- 当前 fold 的离线训练窗口只能来自 5 个训练 run；
- 离线窗口验证和严格在线仿真验证使用同一个被留出的完整 run；
- 在线验证必须在该 run 内保持原始时间顺序，并按既定规则重置 run/segment 状态；不同验证 run 不得首尾拼接为一段连续信号；
- 预处理统计量只能由当前 fold 的 5 个训练 run 估计；
- 每个候选配置必须产生 6 个互斥验证 run 的 out-of-fold 预测，模型选择、停止轮次、阈值和状态机参数只能依据这些预测确定；
- 聚合时可以汇总各 run 的指标或预测记录，但不得让一个 run 的模型状态影响另一个 run；
- 参数全部冻结后，最终模型使用 `0train` 全部 6 个 run 重新训练，预处理统计量也由这 6 个 run 重新估计；
- `1test` 不得参与训练、阈值或决策策略选择；策略冻结后只允许最终模型在其 6 个 run 上分别回放一次。由于既有上游曾解析该文件，不得追加“从未访问的 blind test”声明。

验证清单必须记录每折的训练/验证 run、离线与在线窗口数量，以及所引用母索引和离线视图的文件哈希。

## 3. 统一术语

- `IDLE`：用户未执行目标运动想象；
- `MI`：左手、右手、双脚或舌头运动想象；
- 窗口：一次模型推理所消费的固定长度 EEG 片段；
- 决策时间：窗口结束时间，而不是窗口开始时间；
- MI 事件：数据集中一次带类别标注的完整运动想象区间；
- 触发：系统在 `READY` 状态下输出一次 MI 控制类别；
- 复位：系统输出一次 MI 后，由 Stage 1 的因果 `IDLE` 证据确认用户已回到空闲状态，并重新允许下一次 MI 输出；
- 边界窗口：同时包含 `IDLE` 与 MI 信号的窗口；
- 真值清洗：利用测试标签删除或裁剪边界窗口。

## 4. 官方伪迹排除规则

BNCI2014001 的 MAT 文件以 `artifacts[i]` 标记第 `i` 个 trial 是否被专家判定为含伪迹，但不提供伪迹在 trial 内的精确起止时间。因此，本项目采用保守的 trial 级排除规则。

- `artifacts[i]=1` 时，将对应完整 trial 定义为伪迹排除区间；
- 原始 250 Hz 时钟下，`trial_start_sample=trial[i]-1`，MI 区间为 `[trial_start_sample+500, trial_start_sample+1500)`；
- MOABB `get_data()` 返回的 annotation 已位于 MI 起点，不得再次增加 2 秒；
- 伪迹排除区间为原始时钟下的 `[trial_start_sample, trial_start_sample+1500)`，即 trial 的 0 至 6 秒实验阶段；
- 训练、验证和测试数据使用完全相同的排除规则；
- 任何与排除区间相交的窗口均删除，不参与训练、阈值选择或测试；
- 被排除的 MI 事件不进入事件指标分母，排除区间也不计入有效 `IDLE` 时长；
- 原始 run 不得在排除区间两端直接拼接，也不得压缩原时间戳；
- 所有训练、验证和测试数据都必须先在原始 250 Hz 时钟上切成连续干净 segment，再对每个 segment 独立执行滤波、兼容重采样和窗口化；禁止先滤波完整 run 再删除伪迹窗口；
- 因果滤波器在每个新 segment 都从空状态启动；正式 250 Hz 配置按 4.2 节排除开头 250 点，并在构建清单中逐 segment 记录；
- 每种已冻结的预处理配置只构建一次训练、验证和测试数据，后续实验直接复用，不重复处理原始数据。

首次构建数据集时必须生成清单，记录各 session、run 和类别被排除的 trial 数、窗口数、时长及最终保留量。Subject 1 的官方标记包含训练 session 的 15/288 个 trial 和测试 session 的 7/288 个 trial，独立回归测试必须从源文件重新计算并断言这些数量。

冻结后的数据文件和清单必须保存版本及文件哈希；后续实验只需记录并核对该版本和哈希。如果预处理配置发生变化，则必须生成新的数据版本，不得覆盖旧数据。

正式报告及运行清单必须注明 `artifact_policy=official_trial_exclusion`。伪迹标记只用于定义有效数据 segment，不得作为模型输入或 MI 决策条件；该结果不代表真实部署中的在线伪迹检测能力，真实部署仍需要独立的因果在线伪迹检测器。

### 4.1 原生信号存储

模型取数前先构建原生 250 Hz 的干净 segment 信号存储。该存储只负责通道选择、单位转换和伪迹区间物理隔离，不执行滤波、标准化或重采样。

- 每个任务 run 只保留前 22 个 EEG 通道，3 个 EOG 通道不得写入模型信号文件；
- MAT 中的微伏数值乘以 `1e-6` 转为伏特，并保存为通道优先的 `float32` 数组；
- 每个干净 segment 保存为独立文件，文件内容严格对应其原始 run 的 `[start_native, stop_native)`；
- 不得拼接同一 run 中被伪迹区间分开的 segment，也不得重新编号或压缩原始时间坐标；
- 离线窗口和严格在线窗口只保存元数据，使用 `subject/session/run/segment/start/stop` 从同一份 segment 信号中读取，不重复存储重叠窗口；
- 清单必须记录通道顺序、单位、dtype、源 record、原始坐标、相对文件路径和文件哈希，不得写入本机绝对路径；
- 本层输出不包含 128 Hz 兼容数据；兼容重采样必须在后续预处理层按 segment 独立执行并保存两套坐标。

### 4.2 正式 250 Hz 因果滤波

因果窗口级诊断和严格在线仿真共同使用同一份因果滤波 segment 存储；离线窗口级标准使用单独的零相位预处理分支。

- 采用 `scipy.signal.butter(N=4, [8, 30], btype="bandpass", fs=250, output="sos")` 生成 Butterworth 二阶节；这里 `N=4`，带通传递函数的实际总阶数为 8；
- 使用 `scipy.signal.sosfilt` 沿时间轴单向前向滤波，不执行反向滤波、相位补偿或时间戳前移；
- 每个干净 segment 从全零滤波状态独立启动，禁止跨 run、跨 segment 或跨伪迹排除区间继承状态；
- 滤波以 `float64` 计算并保存为通道优先的 `float32` 伏特数组；清单必须保存完整 SOS 系数、实现方式和上游信号清单哈希；
- 每个 segment 开头 250 点（1 秒）定义为滤波启动排除区间；不足 250 点时排除完整 segment；
- 若某个原生 segment 被启动区间完整覆盖，则它不进入有效评分 segment；但构建清单和在线库存合同必须单独记录这类 segment 的数量和原生采样点数，不得静默删除；
- 启动区间的滤波输出保留用于审计，但任何与它相交的窗口均不得进入训练、验证、阈值选择或正式测试；
- 过滤后不得压缩或平移原始坐标，正式窗口的 `decision_time_seconds` 仍按原始窗口右端计算，报告延迟时不得扣除群延迟；
- 128 Hz MNE 重采样及零相位滤波不属于本节的正式因果配置；零相位离线分支及其因果性标签按 4.3 和第 5 节执行。

### 4.3 实验分支与因果性声明

第一阶段只运行 250 Hz 实验，但后续数据读取、模型输入和评估记录不得写死 500 点窗口或 125 点步长；128 Hz 兼容配置分别使用 256 点和 64 点，并同时保留原生 250 Hz 坐标和模型坐标。

250 Hz 首轮实验固定为：

| 输出 | 训练预处理 | 验证与测试预处理 | 因果性含义 |
| --- | --- | --- | --- |
| 离线窗口级结果 | 零相位 | 零相位 | 传统非因果离线分类能力 |
| 因果窗口级主诊断 | 因果 | 因果 | 在线模型的单窗诊断 |
| 因果窗口级训练消融诊断 | 零相位 | 因果 | 训练非因果、推理因果 |
| 严格在线仿真主结果 | 因果 | 因果 | 训练与推理均因果 |
| 严格在线仿真训练消融 | 零相位 | 因果 | 训练非因果、在线推理严格因果 |

两种训练方式必须使用相同模型结构、随机种子集合、训练窗口 ID 和训练数量；验证与测试的因果输入、窗口顺序和评价规则必须完全相同。训练消融允许模型分别按同一规则使用因果验证预测选择参数，不能使用最终测试 session。

128 Hz 后续实验尽量复用同一矩阵。当前允许直接使用 MNE 重采样进行兼容；只要重采样会访问未来样本，相应输出就必须命名为“128 Hz 兼容窗口结果”或“128 Hz 兼容在线仿真结果”，不得声明为严格因果。

### 4.4 正式 250 Hz 零相位滤波

离线窗口级结果和零相位训练消融使用按 clean segment 独立构建的零相位信号存储，不得对每个 2 秒窗口分别滤波。

- 使用 MNE 1.11.0 的 `filter_data`，参数固定为 `8–30 Hz`、Hamming 窗、`firwin`、`phase="zero"`、`pad="reflect_limited"`；
- MNE 自动规则在 250 Hz 下得到 413 点 FIR；协议显式冻结滤波长度、2 Hz/7.5 Hz 上下过渡带、全部系数及系数哈希；
- 该对称 FIR 的半支撑长度为 206 点（0.824 秒），内部输出会使用当前样本前后各至多 206 点，因此必须标注为非因果；
- 每个 segment 的前 206 点和后 206 点定义为双侧边缘排除区间；不足 413 点的 segment 没有正式有效样本；
- 边缘输出保留用于审计，但正式窗口必须完整位于 `[start_native+206, stop_native-206)` 内；短 segment 的两侧排除区间允许重叠；
- 禁止先对包含官方伪迹的完整 run 做双向滤波再删除伪迹，以免伪迹通过对称 FIR 污染相邻干净信号；
- 本层不执行标准化或重采样；128 Hz 兼容重采样按 4.3 节另行声明因果性。

### 4.5 Fold 专属标准化

标准化参数使用因果与零相位存储共同合法的离线 Stage 1 训练窗口估计。两种训练分支必须使用逐行相同的窗口 ID；Stage 2 使用其中的任务窗口子集，但与 Stage 1 共用标准化参数。

- 每个 fold 只能使用其 5 个训练 run，验证 run 和测试 session 不得参与均值、标准差或窗口筛选；
- 参数按 EEG 通道独立计算，聚合轴为训练窗口维和时间维；重叠窗口中的同一原始采样每出现一次就计数一次，以匹配模型实际输入和原仓库做法；
- 因果窗口只使用同一批训练 run 的因果统计量，零相位窗口只使用零相位统计量；标准化跟随输入域，而不跟随模型身份；
- 因此零相位训练模型在因果验证、测试和在线推理时，输入必须改用由同一训练 run 估计的因果统计量；
- 标准差不得使用固定下限静默裁剪；出现非有限值或非正值必须立即失败；
- 六折选择完成后，最终模型使用全部 6 个训练 run 的共同窗口重新估计两套参数；
- bundle 清单必须保存训练 run、窗口数量、窗口身份哈希、逐通道参数及全部上游文件哈希；checkpoint 必须保存能唯一绑定该 bundle 的合同哈希。验证和测试只能读取这组成套产物并应用既有参数，不得重新拟合；
- 标准化后仍以 `float32` 输入模型；窗口长度从预处理清单读取，不得在数据层写死为 500 点。

## 5. 离线窗口级标准

离线标准用于衡量分类器对干净窗口的基本辨别能力，不代表完整在线系统性能。

该分支按 4.4 节使用零相位滤波，并必须明确标注为非因果离线结果。250 Hz 配置优先实现；原仓库的零相位 128 Hz 处理作为后续兼容结果，不阻塞首轮实验。

固定要求：

- 保留原仓库的两阶段任务：Stage 1 为 `IDLE/Task`，Stage 2 为四类 MI；
- 训练和测试必须保持 session 隔离；
- 离线任务窗口以 MI 起点为锚；每个未标记伪迹且保留完整 4 秒 MI 区间的 trial，按 2 秒窗口和 0.5 秒步长生成 5 个重叠任务窗口；
- 离线 `IDLE` 窗口沿用原仓库的 run 起点网格，并且必须完整位于干净 segment 内且不与任何 MI 区间相交；
- 离线窗口与严格在线仿真窗口分表保存，不得用在线 segment 网格替代上述原仓库离线网格；
- 离线窗口使用左闭右开的 `[start, stop)` 区间；`window` 编号在 Stage 1 的每个 run 内从 0 开始，Stage 2 作为其任务窗口子集保留原 Stage 1 编号，不重新编号；
- 任务窗口、纯 `IDLE` 窗口和边界窗口的定义必须写入运行清单；
- 边界窗口可以不计入离线分类指标，但排除规则必须固定且公开；
- Stage 2 在真实任务窗口上的结果必须标注为真值条件诊断结果；
- 最终五分类结果必须包含 Stage 1 的错误传播；
- 每组窗口级结果必须报告实际参与指标计算的窗口总数，以及各真值类别的窗口数量；
- Stage 2 除总体结果外，应按任务窗口相对 MI 起点的 `0/0.5/1.0/1.5/2.0s` 五个固定偏移分别报告诊断指标；这只是对已有预测分组，不移动窗口或补偿滤波延迟；
- 阈值化结果必须注明阈值及其验证集来源。

离线标准至少保留三组输出：

1. Stage 1 的 `IDLE/Task` 结果；
2. Stage 2 在真实 MI 窗口上的四分类诊断结果；
3. 两阶段级联后的最终五分类结果。

窗口级基础指标冻结如下：

- Stage 1、Stage 2 和两阶段级联必须分开报告；Stage 2 结果继续标注为真实 Task 条件诊断；
- 固定机器标签顺序分别为 `idle/task`、`left_hand/right_hand/feet/tongue` 和 `idle/left_hand/right_hand/feet/tongue`；报告中可对应显示为 `IDLE/Task`、`左手/右手/双脚/舌头`，不得交换整数标签含义；
- 每组结果必须报告 accuracy、balanced accuracy、macro-F1、固定标签顺序的混淆矩阵，以及每类 support、预测数、precision、recall 和 F1；
- balanced accuracy 定义为固定类别 recall 的算术平均；正式汇总缺少任一固定类别时必须失败，逐 run 诊断缺类时 balanced accuracy 与 macro-F1 均返回不可计算，不得只对出现的类别取平均；
- Stage 1 和 Stage 2 还必须由原始 logit 计算未加权 NLL 与多分类 Brier score；NLL 在逐行减去最大 logit 后的坐标中直接计算，不得通过裁剪 softmax 概率改变极端但有限 logit 的损失，也不得用“大数 log-sum-exp 再减真实 logit”的写法引入消减误差；Brier score 定义为每个样本对全部类别的概率平方误差之和再取样本均值；
- 两阶段五分类硬输出固定为：Stage 1 判为 `IDLE` 时输出 `IDLE`，判为 `Task` 时输出 Stage 2 的 argmax 类别；
- 每个 seed 先按窗口身份拼接六个互斥 held-out run 并计算一次汇总指标，再报告三个 seed 的均值和标准差；逐 run 指标单独保留，不替代汇总结果；
- 该指标面板只规定必须计算和报告的量，当前不指定唯一主指标、综合得分或 epoch 选择规则。

### 5.1 250 Hz EEGNet OOF 基线

在不使用最终测试 session 参与拟合或选择的前提下，先在训练 session 内执行六折留一完整 run 的 OOF 训练。正式训练器只能读取已冻结的 session0-only bundle；该 bundle 只含训练 session 窗口、fold 统计量及两域训练 segment，训练器不得自动回访联合索引或测试 session 文件。首批矩阵固定为 `6 folds × 2 stages × 2 training domains × 3 seeds = 72` 条 EEGNet 训练轨迹，使用原生 250 Hz、50 epoch、batch size 64、AdamW 和仅由当前 fold 训练标签计算的 balanced cross-entropy。

因果训练轨迹在因果验证输入上记录 logit；零相位训练轨迹在相同验证窗口上同时记录零相位和因果输入 logit，后者用于“零相位训练、因果推理”消融，不增加训练次数。每个 epoch 均保存原始 OOF logit，不在单个 fold 内独立早停。主指标和统一 epoch 选择规则必须在指标轮冻结后只由训练 session 的六折 OOF 结果确定。完整矩阵、恢复和产物规则见 `docs/baseline_training_matrix.md`。

## 6. 严格在线仿真评估标准

严格在线仿真从每个完整原始测试 run 出发，按照冻结的伪迹清单生成全部有效 segment，并逐 segment 消费其中按时间顺序产生的全部窗口，包括所有 MI/IDLE 边界窗口。除已冻结的伪迹及预处理排除规则外，不得依据测试真值再次删除窗口。

固定要求：

- 每个 segment 以首个有效采样点为滑窗网格起点，从可形成完整窗口的位置开始按固定步长推进；
- 每个 segment 必须先由冻结坐标、窗长和步长生成完整窗口母索引；模型分数与决策轨迹必须逐窗覆盖该母索引，少窗、多窗、重复身份或坐标不一致均直接失败；
- MI 事件分母必须来自独立的 session0 clean event 真值表，不得从 Stage 2 离线任务窗口反推；即使某事件没有任何离线训练窗口，它仍须保留，并仅由在线 segment、窗口和 margin 决定能否评分；
- 真值表与模型 bundle 分开保存并分别绑定哈希。模型推理只读取 signal rows；真值表只能在完整决策轨迹产生后用于事件匹配；
- 历史 v1/v2 结果允许显式使用 `legacy_task_window_reconstruction` 复核；既有 v1 bundle 与独立真值组成 v3，新建 v2 bundle 与独立真值组成 v4。新运行缺少独立真值表时必须失败，不得自动回退；
- 经已冻结排除规则处理后，如果一个连续 clean segment 短于一个完整窗口，则其窗口母索引合法地为空；该 segment 不得被静默删除，其数量、采样点数和时长必须单独报告，并继续计入有效运行时长；
- `subject/session/run/segment/window` 身份和所有原生采样坐标必须为非负整数，其中 subject 从 1 开始；负坐标即使能组成自洽网格也属于无效输入；
- 构建清单必须记录每个 run 的有效 segment、伪迹排除区间、滤波启动排除区间和因长度不足而无法形成窗口的剩余区间；除这些已记录区间外不得删除数据；
- 所有窗口必须保持原始时间顺序；
- 边界窗口必须进入推理流，不得利用真值提前删除；
- 每个窗口只能使用其决策时间及以前的数据和内部状态；
- 每个窗口都必须计算并保存 Stage 1 和 Stage 2 的原始 logit，不能只保存 `argmax` 标签；
- 两个 Stage 使用同一窗口和同一决策时间；Stage 2 是否参与控制输出由后续决策规则决定；
- 推理时不得读取真实状态、真实类别、未来事件或未来数据；
- 真值只允许在推理完成后用于事件匹配和指标计算；
- 每个 run 和 segment 开始时重置滤波器、模型缓存、投票缓存和离散状态机；
- 每个 run 的有效回放时长及排除区间必须记录；
- 正式 250 Hz 配置不执行重采样；滤波和标准化不得访问未来数据；
- 主结果的训练、验证和测试均使用因果预处理；训练消融允许训练使用零相位信号，但验证、测试和在线推理必须使用与主结果完全相同的因果输入；
- 模型和决策器只能保存协议允许的历史状态；
- 检测延迟必须包含窗口积累造成的时间。

严格在线仿真评估不要求按真实时长等待回放，但在每个决策时刻都不得访问未来信息。在线回放输入使用零相位滤波，或使用会访问未来样本的 MNE 直接重采样至 128 Hz 时，相应结果仅用于调试或兼容对照，不得作为正式严格在线仿真结果；该限制不排除“零相位训练、因果输入在线推理”的训练消融。

当前九被试独立真值库存分别包含 `273/270/270/262/262/219/271/264/237` 个事件，总计 2328。它们与历史窗口恢复得到的事件、segment、在线窗口及正式库存哈希逐项一致，因此既有数值结果不变；该等价只说明当前数据没有实际漏事件，不能替代新合同对未来窗口规则变化的防护。

模型原始分数记录至少应包含：

```text
subject_id, session_id, run_id, segment_id
window_index
window_start_native, window_stop_native
window_start_model, window_stop_model
decision_time_seconds
stage1_logits, stage2_logits
```

`window_index` 在每个 segment 内从零开始。窗口采用左闭右开区间；`native` 坐标使用原始 250 Hz run 时钟，`model` 坐标使用当前模型输入时钟。正式配置下二者均为 250 Hz；128 Hz 兼容配置必须同时保留两套坐标。正式 250 Hz 配置的 `decision_time_seconds=window_stop_native/250`，即相对 run 起点取得完整窗口并允许决策的时刻。Stage 1 和 Stage 2 的 logit 长度分别为 2 和 4。

特定决策策略产生的轨迹应与模型原始分数分开保存，至少包含：

```text
subject_id, session_id, run_id, segment_id, window_index
decision_state_before, decision_state_after
emitted_class, transition_reason
```

`decision_state_before/after` 分别表示处理当前窗口前后的决策器状态。`emitted_class=-1` 表示没有输出 MI 指令，`1/2/3/4` 表示输出对应 MI 类别。两状态基线的 `transition_reason` 为空；第 7.3 节的候选策略必须记录与状态转换严格对应的原因。第 7.2 节固定首个硬投票诊断矩阵，第 7.3 节只固定候选态内核，二者都不据此确定最终工作点。

真值时间线应单独保存，不得混入模型输入。

### 6.1 基础事件匹配与在线指标

事件匹配只在完整推理轨迹产生后执行，不得参与决策。正式 250 Hz 首轮采用 `0.5s` 最小 MI 证据 margin，即一个窗口与真实 MI 区间至少重叠 125 个原生采样点，并且窗口右端对应的决策时间不得晚于 MI 终点。窗口起点不要求位于 MI 内；恰好结束于 MI 起点、没有读取 MI 信号的窗口不能匹配事件。

基础事件评估始终使用 BNCI2014001 的原生 250 Hz 坐标，并锁定 500 点窗长、125 点步长和 125 点 margin。未来 128 Hz 模型仍须保留并提交对应的原生坐标；128 Hz 只描述模型输入时钟，不得拿它除原生时间戳或改变事件延迟、有效时长与 FAR 分母。

基础评估器要求调用方显式选择三种互斥模式之一：`stateful_strict` 校验直接输出的 `READY/WAIT_IDLE` 两状态基线；`stateful_candidate` 校验 `READY/TASK_CANDIDATE/WAIT_IDLE` 三状态候选策略；`stateless_diagnostic` 用于无状态决策诊断，状态和转换原因必须为空。三种结果分别报告，不得由评估器自动猜测模式。

- 一个真实事件只有在允许区间内至少存在一个合法决策窗口时才可评分；否则记录为不可评分事件而不是漏检；
- 一个事件最多匹配一个输出，一个输出最多匹配一个事件；在允许区间内按时间最早的实际输出决定事件结果；
- 匹配时不查看输出类别，因此第一次错误类别输出不能被后续正确输出补救；
- 事件结果固定为正确控制、错误类别、`MISS` 或不可评分；事件混淆矩阵行为四个真实 MI 类别，列为四个输出类别加 `MISS`；
- 在 `stateful_strict` 中，`WAIT_IDLE` 的 `emitted_class` 必须为 `-1`，从 `READY` 输出 MI 后必须立即进入 `WAIT_IDLE`，违反者属于无效决策轨迹并直接失败；
- 在基础 `stateful_candidate` 中，MI 指令从 `TASK_CANDIDATE` 提交并立即进入 `WAIT_IDLE`；第 7.8 节允许 Fast-0 在 `READY` 中原子提交，但必须单独记录快速路径原因。候选打开、Stage 1 撤销、候选超时、各路径类别提交和 IDLE 复位都必须记录对应原因；
- 因提前错误复位而在同一真实 MI 内再次产生的输出单独记录为额外事件内输出，不作为正常重复输出指标；
- MI 已开始但尚未满足 0.5 秒 margin 的输出记录为过早输出；真实有效 `IDLE` 时间内的输出记录为 IDLE 误触发。

在线基础结果必须报告：

- 最终正确控制事件率，以及四类正确控制事件率的宏平均；
- 事件内触发率，即正确或错误类别输出的事件数除以可评分事件数；
- 触发后类别准确率，即正确控制事件数除以事件内已触发事件数；
- 漏检率、4×5 事件混淆矩阵、各类事件数和正确数；
- IDLE 误触发次数及每分钟误触发率；分母为有效 segment 总时长减去其中全部真实 MI 时长，不得按窗口数近似；该墙钟时长明确包含零窗口短 segment 和长 segment 末尾不足一个步长的剩余采样，并同时报告这两部分审计量；
- 仅针对正确控制事件的检测延迟 `decision_time-MI_onset`，报告样本数、均值、中位数、Q1、Q3 和 P90；分位数采用 NumPy 默认的线性插值定义，延迟必须与正确事件率同时解释；
- 不可评分事件数、过早输出数和额外事件内输出数作为审计量。

本轮不把上述指标加权成单一总分，也不据此确定多窗聚合、阈值、epoch 或状态机参数。

## 7. 离散控制约束

最终系统采用一次 MI 后必须重新经过 `IDLE` 的离散控制范式。

两个 Stage 在所有时刻都持续预测；下列状态只控制能否发出 MI 指令，不控制运行哪个模型。

最小输出状态定义：

```text
READY -> 允许产生一次 MI 触发
WAIT_IDLE -> 已触发 MI，禁止重复输出，等待确认 IDLE
```

第 7.1、7.2 节直接输出基线的固定原则：

- 仅 Stage 1 满足 Task 条件时不改变输出状态，也不会进入等待 Stage 2 的中间状态；
- 在 `READY` 中，Stage 1 的 Task 证据与 Stage 2 的类别证据联合满足条件时，才输出一次 MI；
- 两组证据可以分别聚合截至当前时刻的历史分数，但不得使用未来窗口；
- 输出 MI 后立即进入 `WAIT_IDLE`，两个 Stage 继续预测，但新的 MI 预测不得产生控制输出；
- 系统不会在输出 MI 后强制假定已经进入 `IDLE`；只有 Stage 1 的因果 `IDLE` 证据满足确认条件后，才返回 `READY`；
- 未能确认 `IDLE` 应记录为复位失败，不得静默强制复位；
- 状态转换参数必须由训练 session 的验证数据确定；
- 无状态结果与加入离散先验的结果必须分开报告。

### 7.1 固定 epoch 50 单窗基线

为给后续多窗聚合和置信度策略提供最低对照，首个在线 OOF 基线固定复用已完成训练的因果 EEGNet，不重新训练：

- `training_budget_epochs=50`、`checkpoint_epoch=50`、`checkpoint_rule=fixed_final_epoch`；
- `epoch_selection_method=none`、`selection_metric=null`；这里的 epoch 50 是预先固定训练预算的最终 checkpoint，不是比较 1–50 epoch 后选出的最佳 epoch；
- 仅使用训练 session 的六折 OOF：第 `k` 个完整 run 只能由未使用该 run 训练的 fold `k` checkpoint 推理；Stage 1/2 必须使用相同 fold、seed 和因果训练域配对；
- 每个连续窗口同时计算 Stage 1 和 Stage 2 logit，不使用置信度阈值或多窗聚合；Stage 1 单窗 `argmax=Task` 时，类别取同一窗口 Stage 2 argmax；
- Stage 1 logit 完全相等时按固定类别顺序判为 `IDLE`；Stage 2 完全相等时取固定顺序中的第一类；
- 无状态诊断在每个 Stage 1 Task 窗都输出一次类别；离散基线在 `READY` 的 Task 窗输出一次并进入 `WAIT_IDLE`，在首个 Stage 1 IDLE 窗只执行复位，从下一窗口起才重新允许输出；
- 每个 segment 均从 `READY` 独立开始，不跨 segment 继承状态；
- 三个 seed 分别报告，再计算描述性均值和总体标准差，不进行 seed 集成或按结果选择 seed；
- 正式候选必须同时包含 seed 42/43/44；单 seed 或任意 seed 子集只能标记为诊断协议，不得沿用正式 protocol ID 或 `CLEAN_COMMIT_FORMAL_CANDIDATE`；
- Subject 1–9 必须分别完成上述评估；跨被试结果先在同一 seed 下对 9 个被试等权宏平均，再报告三个配对 seed 的均值和总体标准差，不得把各被试事件或窗口合并后做微平均；
- 运行器必须先用 checkpoint 重算原训练验证窗口，并与已保存的 epoch 50 OOF logit 核对，再对完整连续窗口推理；两类 logit 和两种决策轨迹均须保存。

该规则只定义“无聚合、无阈值”的基线，不冻结最终系统的复位条件、epoch 选择或模型选择。测试 session 在这些项目全部冻结前继续封存。

### 7.2 联合硬标签多窗口诊断矩阵

首个多窗口基线只在已冻结的逐窗 Stage 1/2 logit 之后增加因果硬投票，不修改模型输入、特征或训练过程。每个窗口恰好产生一票：Stage 1 `argmax=IDLE` 时投 `0`；Stage 1 `argmax=Task` 时投同窗 Stage 2 的 `argmax` 类别 `1..4`。argmax 平局继续采用固定类别顺序，因此 Stage 1 平局投 `0`。

固定网格如下：

```text
N=2: K=2
N=3: K=2,3
N=4: K=3,4
N=5: K=3,4,5
```

- `N` 是当前状态内最近硬标签缓存的最大窗口数，必须收满 `N` 票后才允许转换；
- `K` 同时作为 MI 输出和 IDLE 复位的最低票数，并且始终满足 `K>N/2`，所以不会出现两个类别同时过阈值；
- `READY` 中任一 MI 类别 `1..4` 获得至少 `K` 票时输出一次该类，立即进入 `WAIT_IDLE` 并清空缓存；
- `WAIT_IDLE` 中标签 `0` 获得至少 `K` 票时只返回 `READY`，该窗口不得同时输出 MI，并再次清空缓存；
- 每个 segment 开始时状态重置为 `READY` 且缓存为空，缓存不得跨 segment 继承；
- 缓存只包含当前及过去窗口，不得读取未来窗口；所有窗口继续保留原始 logit，投票标签和完整状态轨迹另行保存；
- `N=1,K=1` 不重复生成，直接引用第 7.1 节冻结的单窗离散基线作为矩阵锚点；
- 本矩阵不使用 softmax 概率、置信度阈值、logit/特征均值、TempDens 或真值辅助决策。

九被试和三个配对 seed 的全部 cell 必须完整报告，继续采用同一 seed 内被试等权宏平均。该矩阵是描述性诊断：不得在同一批 OOF 结果中选择某个 cell，再把该 cell 的数值称为无偏验证性能。最终工作点及更复杂的软分数或特征聚合方法须在后续轮次单独冻结选择规则。

0.5 秒事件匹配 margin 已冻结。硬投票矩阵必须与正确事件率、IDLE 误触发率和检测延迟共同报告；最终置信度阈值及其他聚合方法仍待后续轮次确定。

### 7.3 可撤销候选态决策内核

候选态策略把三个职责分开：Stage 1 高召回地打开并维持候选区间，Stage 2 只负责高精度提交四类 MI，`WAIT_IDLE` 只负责禁止重复输出和等待复位。本轮只固定状态内核和轨迹校验，不固定 EWMA、logit/probability 聚合、阈值、连续确认窗数或最终工作点。

```text
READY -> TASK_CANDIDATE -> WAIT_IDLE -> READY
              |
              +---------> READY   （撤销或超时，不输出）
```

基础内核要求调用方为每个窗口因果地产生四项抽象证据：`task_on`、`task_hold`、`stage2_commit_class` 和 `idle_reset`。其中 `stage2_commit_class=-1` 表示尚无类别满足提交条件，`1..4` 对应左手、右手、双脚和舌头。本轮不规定这些证据如何由 logit 得到。第 7.8 节另加 `fast_commit_class` 字段，且不改变这四项慢通道证据。

- 基础内核在 `READY` 中只有 `task_on=true` 才进入 `TASK_CANDIDATE`，该窗口不得输出命令；第 7.8 节的 Fast-0 是唯一显式例外；
- Stage 2 原始分数始终保存。进入候选态时慢通道 Stage 2 历史必须清零，只有后续窗口的证据进入慢通道；第 7.8 节快速缓存使用开门窗时，不得写入或改变该慢通道历史；
- `TASK_CANDIDATE` 中的固定优先级为：`task_hold=false` 时由 Stage 1 撤销并返回 `READY`；否则若 Stage 2 给出类别则提交一次并进入 `WAIT_IDLE`；否则在显式的最大候选窗数耗尽时超时返回 `READY`；
- 最后一个允许的候选窗口仍先给 Stage 2 提交机会，未提交才判超时；`max_candidate_windows` 必须由调用方显式传入且为正整数，本轮不提供默认值或正式取值；
- `WAIT_IDLE` 中所有 MI 类别证据均不能产生输出，只有 `idle_reset=true` 才返回 `READY`，复位窗口不得同时重新开门；
- 每个 segment 从 `READY` 重新开始，未完成候选区间记录为 `segment_end_unresolved`，不得跨 segment 继承；
- 运行时必须逐窗保存四项基础输入证据、候选态年龄、状态前后、转换原因和输出类别；启用第 7.8 节时还须保存快速提交证据及路径诊断，以便验证实现与声明一致。

除第 6.1 节的事件指标外，`stateful_candidate` 还必须报告候选打开次数及每有效分钟打开率、提交转化率、Stage 1 撤销率、超时率、segment 末未完成率，以及已完成候选区间的时长分布。提交、Stage 1 撤销、超时和未完成四项比例均以全部候选打开次数为分母，未完成候选也保留在该分母中；没有候选打开时比例记为 `null`。已完成候选时长从打开候选窗口的决策时刻量到提交、撤销或超时窗口的决策时刻，segment 末未完成候选不进入该时长分布。若一个 `MISS` 的合法事件窗口内出现候选超时，记录为“超时相关 MISS”；其比例以全部可评分 `MISS` 数为分母，没有可评分 `MISS` 时记为 `null`。该量只表示时间关联，不得表述为超时造成 MISS。真值仍只允许在完整推理轨迹生成后用于这些诊断统计。

### 7.4 候选态 logit 策略诊断矩阵

首个候选态分数矩阵固定复用第 7.1 节的 session 0 OOF logits，不重新训练，也不访问测试 session。参数是在查看同一 OOF logits 的无标签尺度分布后冻结的探索值，因此本矩阵只用于理解机制和提出下一轮工作点选择规则，不是无偏性能估计。八个 cell 必须全部报告，不得只保留结果较好的 cell。

所有 cell 共享以下定义：

- Stage 1 基础分数为 `sigmoid(task_logit-idle_logit)`；`ewma_probability` cell 是先转概率再做 EWMA，其余 EWMA cell 是先平滑二分类 margin 再转概率；
- Stage 2 每窗 logits 先减去本窗四类均值，消除公共平移；均值和 EWMA 只聚合进入候选态后的窗口，开门窗不进入 Stage 2 历史；
- Stage 1 的 EWMA、rolling 和差分在每个 segment 重置；Stage 2 历史还会在每次候选打开及提交、撤销、超时后清空；
- EWMA 以首个可用分数初始化，rolling 在不足指定窗数时使用当前全部因果前缀，不做未来填充；
- `task_on/task_hold/idle_reset` 的共同阈值为 `0.50/0.30/0.20`；打开、维持和提交使用 `>=`，IDLE 复位使用 `<=`；
- 最大候选长度统一为开门后的 8 个窗口，即当前 0.5 秒步长下最多 4.0 秒；最后一窗仍先尝试提交，再判超时；
- 默认 Stage 2 提交条件为聚合后 top-1 概率至少 `0.55` 且 top-1 与 top-2 概率差至少 `0.15`；高精度 cell 改为 `0.70/0.30`；
- 类别稳定数按开门后连续的单窗 raw top-1 计算，并要求该类别等于当前聚合 top-1；二阶曲率固定使用最近三个候选窗各自的单窗 raw Stage 2 softmax 概率向量，计算其二阶差分 L2 范数，不使用均值或 EWMA 后的聚合概率，少于三窗时不能通过曲率条件；
- Stage 1 突降撤销定义为过滤后 Task 概率差分 `<=-0.20`，即使同窗 Stage 2 已满足提交条件也由 Stage 1 撤销优先；
- 所有阈值判断和时间变换只能使用当前及历史窗口；完整轨迹生成后才允许读取事件真值计算第 6.1、7.3 节指标。

固定八个并列 cell：

| cell | Stage 1 | Stage 2 候选内证据 | 最少候选窗 / 额外条件 |
|---|---|---|---|
| `raw_current` | 当前 margin | 当前中心化 logits | 1 |
| `stage1_ewma` | margin EWMA，`alpha=0.5` | 当前中心化 logits | 1 |
| `stage2_mean` | 当前 margin | 中心化 logits 均值 | 2 |
| `dual_ewma` | margin EWMA，`alpha=0.5` | 中心化 logits EWMA，`alpha=0.5` | 2 |
| `dual_ewma_drop_abort` | 同上，加 Task 概率突降撤销 | 中心化 logits EWMA | 2 |
| `rolling_stable` | 最近 3 窗 margin 均值 | 中心化 logits 均值 | 3，raw 类别连续 2 窗稳定 |
| `probability_curvature` | Task 概率 EWMA，加突降撤销 | 中心化 logits 均值 | 3，稳定 2 窗且概率曲率 `<=0.35` |
| `dual_ewma_high_precision` | margin EWMA | 中心化 logits EWMA | 2，稳定 2 窗且采用 `0.70/0.30` 提交阈值 |

每个 cell 除基础事件指标外，还须并列报告候选打开率、提交转化率、Stage 1 撤销率、超时率、未完成率、候选时长及超时相关 MISS。特征空间时间变化尚未进入本矩阵；只有先证明重新提取隐藏特征时的 logits 与冻结 logits 对齐，才能作为独立后续矩阵加入。

### 7.6 MI 提交与 `WAIT_IDLE` 复位析因矩阵

为避免把“类别提交更严格”和“输出后复位更慢”混成同一参数，本矩阵复用第 7.1 节冻结的九被试、三 seed、session 0 OOF logits，不重新训练、不访问测试 session，并把两个职责显式做成笛卡尔积。共同骨架固定为 `dual_ewma_drop_abort`：Stage 1/2 EWMA 均取 `alpha=0.5`，`task_on/task_hold=0.50/0.30`，Stage 2 gap 为 `0.15`，最少候选窗数为 2，最大候选窗数为 8。

- MI 提交轴只改变 Stage 2 聚合后 top-1 概率阈值：`0.55/0.625/0.70`；gap、稳定窗数、候选长度和 Stage 1 条件保持不变；
- 复位轴只改变过滤后 Stage 1 Task 概率阈值 `0.20/0.30/0.40`，分别要求连续 `1/2` 个窗口满足；连续计数只在 `WAIT_IDLE` 内累积，任一失败窗口、复位或 segment 边界均清零；
- `task_hold` 只决定候选态是否继续，`idle_reset` 只决定命令发出后是否重新待命，因此不再强制 `idle_reset<=task_hold`；为避免复位后立即重开，仍必须满足 `idle_reset<task_on`；
- 复位窗口只允许 `WAIT_IDLE->READY`，不得在同窗重新打开候选；窗口步长为 0.5 秒，连续两窗只代表多等待一步，且两窗共享 1.5 秒输入，不得解释为两份独立证据；
- 18 个 cell 必须完整并列报告。解释提交效应时固定复位 profile，解释复位效应时固定提交 profile，不得横跨两个同时变化的 cell 作单因素因果表述；
- `top_probability` 和 Stage 1 Task 概率均为模型派生分数，未做概率校准，不得解释为真实正确概率。

除第 6.1、7.3 节指标外，本矩阵逐 cell 报告 `WAIT_IDLE` 完成及 segment 末删失时长、匹配命令复位相对真实 MI offset 的分布、提前复位率、事件首个可计分窗仍锁定率、整个事件完全锁定率，以及 IDLE 误指令中的“MI 内打开候选、MI 后才提交”和其他误指令。事件真值只能在完整轨迹生成后参与这些归因，不得改变在线状态。`0.55/0.20/1窗` cell 必须逐 seed、逐被试复现第 7.4 节 `dual_ewma_drop_abort` 的正式事件指标，作为实现回归锚点。

### 7.7 正确候选提交延迟来源诊断

本诊断固定使用第 7.6 节的历史回归锚点 `c055_r020_l1`，不重新训练、不访问测试 session，也不从第 7.6 节结果中选择新工作点。基线在 Stage 1 开门窗不更新 Stage 2 候选 EWMA，且只在开门后的第二个候选窗允许提交；在 0.5 秒步长下，这形成开门后 1.0 秒的结构性提交下界。

正式端到端延迟仍按 `decision_sample/250-MI_onset_sample/250` 计算。因果滤波引起的分数响应滞后已经包含在该时间中，不移动时间戳、不扣除固定群延迟；滤波、模型和状态机的真实墙钟计算耗时尚未测量，不能混入本轮结果。诊断只准确拆分 Stage 1 开门时机、基线候选驻留时间、固定两窗等待及其后的额外等待。

完整基线轨迹生成后，允许使用真值计算下列只读 oracle：

- 原始 Stage 2 单窗 top-1 首次等于真值，以及同时满足冻结 `0.55/0.15` 概率与 gap 的时刻；
- 从 Stage 1 开门窗开始计算 Stage 2 `alpha=0.5` EWMA 后，正确 top-1 首次出现的时刻；
- 同一开门窗起算 EWMA 在允许开门窗立即提交的 `min1` 条件下，以及把开门窗计作第一窗的 `min2` 条件下，首次满足正确类别置信度的时刻。

所有 oracle 窗口仍必须满足正式 0.5 秒事件证据 margin；开门后的窗口仍须满足原 Stage 1 `task_hold`，不得跨越候选退出或 segment 边界。真值只能选择和解释已经产生的分数，不得修改原始决策、候选边界或复位状态。

oracle 会跳过更早出现的错误类别，因此只表示信号和现有模型中可能存在的乐观上限，不是可部署策略，也不得报告为正确事件率、FAR 或新系统成绩。为暴露这一乐观偏差，必须另行在每个真值定义的事件局部范围内，在不读取 `true_class` 的情况下选择第一个置信度 crossing，再用类别真值标注其正确性；至少报告 crossing 覆盖率、首 crossing 类别准确率、正确事件率、在基线正确事件中的提前且正确比例，以及 truth-aware oracle 此前已出现错误 crossing 的比例。

该量只能称为“使用真值事件边界事后裁剪的 class-label-free 首 crossing”：类别选择不看类别真值，但事件 onset/offset 和 0.5 秒 margin 仍来自真值。它也没有在全部 IDLE 候选上重放提前提交后的后续状态变化，所以不是完整反事实策略成绩。

结果还必须同时报告原基线正确事件率、逐事件配对 headroom、正确类别的 MI 后溢出数量、其中基线 MISS 事件的可救回比例、首窗及整事件 `WAIT_IDLE` 锁定率。已经正确或错误触发的事件即使后来又产生正确类别溢出，也不得计入“救回 MISS”的分母。只有同一个基线正确事件在 oracle 下仍有不晚于原提交的正确证据，才进入配对提前量；不得通过丢弃慢事件单独降低延迟中位数。

### 7.8 Fast-0、Fast-1 与慢通道兜底矩阵

本矩阵把第 7.7 节的事后诊断改成完整在线轨迹重放，但仍只复用九被试、三 seed 的 session 0 OOF logits，不重新训练、不访问测试 session。历史 `c055_r020_l1` 作为慢通道锚点；快速通道不改变 Stage 1 EWMA、`WAIT_IDLE` 复位、最大候选长度或事件匹配规则。

- **Fast-0**：当 `READY` 中本窗满足 Stage 1 开门条件和更严格的单窗 Stage 2 条件时，本窗直接输出类别并原子地转入 `WAIT_IDLE`，不先产生一个只有中间状态、没有输出的候选窗口；
- **Fast-1**：Fast-0 未通过时照常打开候选。开门窗与紧接的下一窗使用独立的两窗 Stage 2 EWMA，且两窗 raw top-1 类别必须一致；只在该下一窗检查一次，失败后清空快速缓存。底层状态机和形式评估器都必须拒绝第二个及更晚候选窗伪装成 Fast-1；
- **慢通道兜底**：仍从开门后的窗口开始建立独立 Stage 2 EWMA，开门窗不进入慢通道历史。Fast-0/Fast-1 均未提交时，后续每窗的慢通道分数、两窗最小等待、Stage 1 撤销、八窗超时和复位规则必须与历史锚点逐窗完全一致；
- 开门后的首窗优先级固定为 Stage 1 撤销、Fast-1、慢通道、超时。快速缓存与慢通道缓存不得共用，避免 Fast-1 失败后改变兜底结果；
- Fast-0、Fast-1、慢通道必须使用不同的 `transition_reason`，并分别报告命令占比、正确事件、触发后分类准确率、IDLE 误指令和正确提交延迟；三条路径的计数必须严格相加回正式评估器总数，条件指标还必须保存每个 seed 的有效被试数；
- 逐窗轨迹除状态、原因和通过标记外，必须保存 raw、Fast-1、慢通道的 top-1 类别，以及 IDLE 原始复位条件和连续计数；
- 相对无快速通道锚点，还须逐事件配对报告：锚点正确但当前不正确、锚点 MISS 被救成正确，以及两者都正确时是否更早。配对比较不得改变事件库或可评分条件。

固定七个并列 cell：无快速通道锚点；Fast-0、Fast-1 和二者联合的 `balanced` 版本；Fast-0、Fast-1 和二者联合的 `strict` 版本。Fast-0 共同要求 Stage 1 Task 概率至少 `0.50`、相对前窗增量至少 `0.03`；Fast-1 共同要求开门窗及下一窗的 Stage 1 Task 概率均至少 `0.60`，Stage 2 EWMA `alpha=0.5`。`balanced` 的 Stage 2 top-1/gap 为 `0.90/0.80`，`strict` 为 `0.95/0.90`。这些阈值是在查看同一 session 0 OOF 分数尺度及事后归因后冻结的探索网格，七个 cell 必须全部报告，不能从中挑出一个数值后称为无偏验证结果。

### 7.9 决策部件 Oracle 与 Stage 2 证据上限

本诊断回答“当前正确事件率的损失主要发生在哪个决策职责，以及冻结 Stage 2 logits 中最多还保留多少正确类别信息”。它只复用九被试、三个配对 seed 的 session 0 OOF logits，不训练新模型、不访问测试 session。三种子的 logits 和纯模型 `c055_r020_l1` 锚点轨迹必须全部先生成，随后才允许加载独立事件真值侧车。全模型 cell 必须逐窗复现历史锚点的输出、状态和转换原因；不满足时整次运行失败。

对 Stage 1、MI 提交和 `WAIT_IDLE` 复位三个职责分别使用模型证据或真值 Oracle，形成完整 `2×2×2` 析因矩阵：

- **Stage 1 模型轴**沿用 `alpha=0.5` EWMA、`task_on/task_hold=0.50/0.30` 和 `0.20` 突降撤销；**真值轴**把每个满足正式 0.5 秒 margin 的事件窗口视为 Task，其余视为 IDLE。开门窗仍不进入慢通道 Stage 2 历史，不借 Oracle 偷改两窗最小等待；
- **提交模型轴**在候选内首次同时满足 Stage 2 `top_probability>=0.55`、`gap>=0.15`、最少两窗，并且当前单窗 raw top-1 与候选 EWMA 聚合 top-1 一致时，立即提交该聚合 top-1；最后一项对应历史 `stage2_stable_windows=1`，不是可以省略的恒真条件。**真值轴**仍要求同一个模型 crossing 全部合法，但可以跳过错误 crossing，只能在模型聚合 top-1 等于事件真类时提交，不能凭真值虚构类别或分数；
- **复位模型轴**沿用 `WAIT_IDLE` 中 Stage 1 Task 概率 `<=0.20` 的单窗确认；**真值轴**在命令后的第一个 `window_stop_sample>event.offset_sample` 决策窗复位。这里判断的是决策时刻已经进入真实 IDLE，不声称整个两秒输入窗均为纯 IDLE；若命令本来就发生在 IDLE，则最早也只能在下一窗口复位；
- 真值提交轴必须在每个完整 segment 上联合优化，不能把逐事件最优点独立拼接。目标依次为：正确且唯一匹配的事件数最多、正确事件总延迟最小、命令数最少、提交窗口按时间更早；所有选择完成后仍须交给同一个正式事件评估器复算；
- 每个 segment 独立从 `READY` 开始。真值提交后的轨迹仍真实经过 `WAIT_IDLE`；复位过早或过晚对后续事件的阻塞与重开影响不能绕过。

完整八格同时用于正确事件率的 Shapley 描述性分解，把三个部件的交互增益按所有替换顺序平均分配。三个分量必须严格加回“全真值 cell 减全模型 cell”的差，但它们不是可识别的因果效应，允许出现负贡献。

另对每个正式可评分事件计算三层 Stage 2 标签知情证据上限：

1. 任一合法单窗 raw top-1 等于真类；
2. 从任一合法事件窗起算的 `alpha=0.5` 因果 EWMA，在某个合法前缀末端 top-1 等于真类；
3. 截至当前合法前缀，存在一组非负且和为 1 的窗口权重，使真类 logit 不低于其他三类。第三项用线性规划求最大最小类间 margin，允许标签知情的并列 top-1，因此是更宽松的纯信息上限。

证据上限必须报告覆盖率、相对锚点正确事件率的最大差、锚点 MISS/错类中具有正确证据的比例和最早证据延迟。它不包含可部署的开门、提交、复位或 IDLE 抑制规则。尤其是真值提交轴的 FAR 为零或降低主要是因为它按定义拒绝 IDLE 和错误类别 crossing，不能报告为系统 FAR 改进，也不能用来选择策略。所有结果继续按同一 seed 内九被试等权宏平均，再汇总三个配对 seed；不得合并事件做跨被试微平均。

### 7.10 LD-GRU-v1 候选态学习型提交器

LD-GRU-v1 只替换 `TASK_CANDIDATE` 内的 Stage 2 提交条件。Stage 1 的开门、维持、突降撤销以及提交后的 `WAIT_IDLE` 复位继续固定为 `dual_ewma_drop_abort` 的 `alpha/on/hold/drop/reset=0.5/0.50/0.30/0.20/0.20`；最大长度仍为开门后 8 窗。Stage 1/2 EEGNet 均冻结，只读取 epoch 50、session 0、run-level OOF 连续 logits，不访问 session 1。

每个候选窗的 12 维 token 依次为：Stage 1 当前 Task-IDLE margin、margin EWMA 后的 Task 概率及其一阶差分、四维中心化 Stage 2 logits、相对同 segment 上一正式窗口的四维一阶差分、候选年龄除以 8。前 11 维只用当前决策模型训练被试拟合总体 mean/std；小于 `1e-6` 的常量 std 置为 1，年龄不标准化。开门窗口以年龄 0 写入 GRU hidden，但 `q0` 强制为 0；最早只允许紧接的下一窗提交，不含 Fast-0。Stage 1 撤销窗先退出且不送入 GRU。固定优先级为 Stage 1 撤销、GRU 提交、候选超时。

模型固定为 `GRUCell(12,8)`、`Linear(8,1)` stop head 和 `Linear(8,4)` 类别残差头，共 573 个参数。stop bias 初始化为 -3；类别残差头权重和 bias 初始化为 0。并列训练两个消融：`stop_only` 的类别始终取当前中心化 Stage 2 logits 的 argmax，残差头保留在结构中但冻结为 0，共 537 个可训练参数；`stop_residual` 同时学习停止和四类残差，共 573 个可训练参数。

训练候选由不含 Stage 2 提交的固定 Stage 1 生成器在完整正式窗口母索引上切出，保留 IDLE、边界、MI 内和 MI 后候选。开门窗进入序列但监督掩码恒为 0。对后续每个“时间×类别”调用与正式评估器共用的 `classify_counterfactual_first_commit()`；只有其结果为 `correct` 才进入集合式正确掩码。单 token 候选仍进入库存统计和训练被试的标准化统计，但因其损失恒为零，不进入梯度 batch。

若候选内存在合法正确点，损失为首次提交分布落在任一正确点的总概率之负对数；否则为整段不提交概率之负对数。每个 batch 的正、负候选分别求均值后各占 0.5。第一版不加人工延迟权重、错误类型权重、强化学习或辅助任务。优化固定为 AdamW，`lr=1e-3`、`weight_decay=1e-4`、batch 64、最多 200 epoch、梯度裁剪 1.0、early-stopping patience 20；决策 seed 为 `1000+base_seed`。

评价采用外层九被试 LOSO。对每个 outer subject 和每个 base seed，决策模型的训练、标准化、epoch 与阈值选择均不得使用该 outer subject。内层在其余八人中循环七人训练、一人验证。为避免重复计算，同一无序留出对 `{A,B}` 只训练一条相同的七被试梯度轨迹，但 A、B 分别维护独立 patience；任一端点首次达到 patience 后立即永久冻结其最佳 checkpoint，即使另一端使训练继续、该端后来再次改善也不得改写。两端均冻结或达到 200 epoch 才结束共享计算，因此严格对应 `outer=A,val=B` 与 `outer=B,val=A` 两个互不泄漏的有序 fold。每个 seed/消融是 36 条唯一梯度轨迹、72 个有序内层验证 fold。最终 epoch 取某 outer 的八个内层最佳 epoch 中位数并按 half-up 取整，再在其余八人上从头固定 epoch 训练。

stop threshold 只在八个内层 held-subject 完整在线回放上扫描 `0.05,0.10,...,0.95`。先要求八人等权平均 FAR 不高于各自 `dual_ewma_drop_abort c055_r020_l1` 的等权平均 FAR，再依次按正确事件率最大、FAR 最小、正确延迟 P90 最小、阈值更高确定唯一工作点。outer subject 只在 epoch、阈值、标准化和模型均冻结后运行一次正式回放。

最终至少并列报告单窗两状态基线、`n5_k3` 硬投票、`dual_ewma_drop_abort c055_r020_l1` 和两个 LD-GRU 消融。学习策略仍使用 `stateful_candidate` 正式评估器；新增提交原因固定为 `learned_gru_commit`。每窗保存 raw/标准化 token、hidden、stop logit/score、中心化 logits、类别修正、最终 logits、是否真正消费 token、候选年龄、状态、输出和转换原因。未消费 GRU 的窗口仍保存有限的 11 维基础流特征、中心化 Stage 2 logits 和清零 hidden；年龄用 `-1`、stop/correction 用 0 作为显式中性占位，并由 `gru_consumed=false` 区分，禁止使用 NaN 掩盖未运行路径。提交、撤销、超时、复位或 segment 边界均清空 hidden。阈值扫描使用 CPU 单步回放可以减少微型模型的 GPU 启动开销，但所有反向训练必须使用已记录的 CUDA GPU。

## 8. 复现与审计要求

每次正式评估必须保存运行清单，至少记录：

- Git commit；
- 数据集、被试、session 和 run；
- 冻结数据文件、构建清单版本及文件哈希；
- 预处理、窗口和步长参数；
- 数据划分及验证参数来源；
- checkpoint 路径及文件哈希；
- 阈值、状态机和事件匹配配置；
- 随机种子与运行环境；
- 预测文件、指标文件和日志路径。

基础评估器还必须在结果中写出 segment、完整窗口母索引、真值事件和决策轨迹的规范化 SHA-256；正式运行时这些哈希必须与冻结构建清单及实际输入一致，以便发现上游静默删窗或替换时间线。

后续决策策略复用历史正式 logits 时，必须核对历史清单自身冻结的协议 ID、生成 commit、源码哈希合同及全部产物哈希，不得把历史清单中的哈希替换为当前值。历史生成源码不要求与后续策略的当前源码字节相同，否则评估器的合法扩展会错误地使冻结 logits 失效；新产生的运行仍必须默认绑定运行当时的当前源码。

任何协议字段缺失的结果只能用于调试，不得作为正式可比较结果。

## 9. 后续待冻结项目

以下内容仍必须通过独立轮次讨论、实现和审核：

1. 是否把第 7.5 节特征门控扩展至九被试，以及只由训练数据确定阈值的规则；
2. 如何从第 7.2、7.4 节描述性矩阵冻结 Stage 1/2 阈值、连续确认和最终工作点；
3. 如何依据第 7.6 节析因矩阵冻结最终提交与复位工作点，以及是否进一步改变最大候选时长；
4. 如何依据第 7.8 节完整快速通道矩阵冻结候选方案，并在独立于阈值探索的数据上评价；
5. 是否需要重新设计 Fast-0 的 Stage 1 条件，或只保留更稳定的 Fast-1；
6. epoch、随机种子、集成与多指标模型选择规则；
7. 128 Hz 兼容预处理协议；
8. BNCI2014001 端到端验收条件。
