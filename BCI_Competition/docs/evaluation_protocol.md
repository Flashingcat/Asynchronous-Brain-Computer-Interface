# 异步运动想象评估协议

## 1. 文档目的

本文档规定本项目评估代码必须遵守的顶层协议，用于避免数据泄漏、时间穿越和指标口径混用。

当前验证范围固定为 BNCI2014001 的 Subject 1。模型结构不是本评估工作的研究对象；原仓库的两阶段分类器作为首个基线。

本协议的所有正式结果均基于官方 trial 级伪迹标记清洗后生成的冻结数据；该前提不代表系统已经实现在线伪迹检测。

本协议区分两种正式结论和一种中间诊断，报告结果时不得混称：

1. 离线窗口级结果；
2. 因果窗口级诊断结果；
3. 严格在线仿真结果。

其中第 2 项使用在线模型的因果输入和 logit，但不运行连续决策器，用于分离“单窗分类能力”和“时间聚合、阈值及状态机”的影响，不作为独立系统结论。

## 2. 固定数据边界

- 数据集：BNCI2014001；
- 被试：Subject 1；
- 训练来源：`0train` session；
- 最终测试来源：`1test` session；
- 每个 MAT 文件按原始顺序选取恰好包含 48 个 trial 的 6 个记录，并依次映射为任务 run 0 至 5；无 trial 的眼动校准记录不得作为 `IDLE`。Subject 1 的任务记录对应 `data[3]` 至 `data[8]`，其他被试不得假定相同固定下标；
- 分类输入只使用前 22 个 EEG 通道，3 个 EOG 通道不得作为分类特征；
- 正式协议使用原生 250 Hz，不重采样；2 秒窗口包含 500 点，0.5 秒步长包含 125 点；
- 兼容配置允许沿用 MNE 重采样至 128 Hz，得到 256 点窗口和 64 点步长，但必须标为非严格因果的兼容结果；
- 最终测试必须将 `1test` 的 6 个 run 分别从头到尾回放；
- 每个 run 开始时重置因果滤波缓存、历史分数缓存和决策状态，禁止带入上一 run 的历史信息；
- run 之间保持模型参数和训练得到的标准化参数不变；
- 测试 session 不得参与模型选择、停止轮次选择、阈值选择或连续确认参数选择，以防止测试信息泄漏；
- 需要调节的参数只能由训练 session 内部的验证集确定；验证划分以完整任务 run 为最小单位，同一 run 的 trial、segment 和窗口不得跨集合，禁止随机按窗口或 trial 划分；
- 由数据估计的预处理参数——当前包括每个 EEG 通道的均值和标准差——只能由当前 fold 的训练 run 计算；
- 验证集和测试集只能应用训练得到的预处理参数，不得参与估计；这些参数应随 checkpoint 保存。

### 2.1 训练 session 内部验证划分

`0train` 的 6 个任务 run 采用固定的六折留一 run 验证，不使用随机划分：fold 编号等于验证 run 编号；当前 fold 以该 run 为验证集，其余 5 个 run 为训练集。每个 run 恰好作为验证集一次，因此该划分没有随机种子。

- 当前 fold 的离线训练窗口只能来自 5 个训练 run；
- 离线窗口验证和严格在线仿真验证使用同一个被留出的完整 run；
- 在线验证必须在该 run 内保持原始时间顺序，并按既定规则重置 run/segment 状态；不同验证 run 不得首尾拼接为一段连续信号；
- 预处理统计量只能由当前 fold 的 5 个训练 run 估计；
- 每个候选配置必须产生 6 个互斥验证 run 的 out-of-fold 预测，模型选择、停止轮次、阈值和状态机参数只能依据这些预测确定；
- 聚合时可以汇总各 run 的指标或预测记录，但不得让一个 run 的模型状态影响另一个 run；
- 参数全部冻结后，最终模型使用 `0train` 全部 6 个 run 重新训练，预处理统计量也由这 6 个 run 重新估计；
- `1test` 始终保持封存，只允许最终模型在其 6 个 run 上分别回放一次。

验证清单必须记录每折的训练/验证 run、离线与在线窗口数量，以及所引用母索引和离线视图的文件哈希。

## 3. 统一术语

- `IDLE`：用户未执行目标运动想象；
- `MI`：左手、右手、双脚或舌头运动想象；
- 窗口：一次模型推理所消费的固定长度 EEG 片段；
- 决策时间：窗口结束时间，而不是窗口开始时间；
- MI 事件：数据集中一次带类别标注的完整运动想象区间；
- 触发：系统在 `READY` 状态下输出一次 MI 控制类别；
- 复位：系统输出一次 MI 后，由 Stage 1 的因果 `IDLE` 证据确认用户已回到空闲状态，并重新允许下一次 MI 输出；
- 边界窗口：同时包含 `IDLE` 与 MI 信号的窗口；
- 真值清洗：利用测试标签删除或裁剪边界窗口。

## 4. 官方伪迹排除规则

BNCI2014001 的 MAT 文件以 `artifacts[i]` 标记第 `i` 个 trial 是否被专家判定为含伪迹，但不提供伪迹在 trial 内的精确起止时间。因此，本项目采用保守的 trial 级排除规则。

- `artifacts[i]=1` 时，将对应完整 trial 定义为伪迹排除区间；
- 原始 250 Hz 时钟下，`trial_start_sample=trial[i]-1`，MI 区间为 `[trial_start_sample+500, trial_start_sample+1500)`；
- MOABB `get_data()` 返回的 annotation 已位于 MI 起点，不得再次增加 2 秒；
- 伪迹排除区间为原始时钟下的 `[trial_start_sample, trial_start_sample+1500)`，即 trial 的 0 至 6 秒实验阶段；
- 训练、验证和测试数据使用完全相同的排除规则；
- 任何与排除区间相交的窗口均删除，不参与训练、阈值选择或测试；
- 被排除的 MI 事件不进入事件指标分母，排除区间也不计入有效 `IDLE` 时长；
- 原始 run 不得在排除区间两端直接拼接，也不得压缩原时间戳；
- 所有训练、验证和测试数据都必须先在原始 250 Hz 时钟上切成连续干净 segment，再对每个 segment 独立执行滤波、兼容重采样和窗口化；禁止先滤波完整 run 再删除伪迹窗口；
- 因果滤波器在每个新 segment 都从空状态启动；正式 250 Hz 配置按 4.2 节排除开头 250 点，并在构建清单中逐 segment 记录；
- 每种已冻结的预处理配置只构建一次训练、验证和测试数据，后续实验直接复用，不重复处理原始数据。

首次构建数据集时必须生成清单，记录各 session、run 和类别被排除的 trial 数、窗口数、时长及最终保留量。Subject 1 的官方标记包含训练 session 的 15/288 个 trial 和测试 session 的 7/288 个 trial，独立回归测试必须从源文件重新计算并断言这些数量。

冻结后的数据文件和清单必须保存版本及文件哈希；后续实验只需记录并核对该版本和哈希。如果预处理配置发生变化，则必须生成新的数据版本，不得覆盖旧数据。

正式报告及运行清单必须注明 `artifact_policy=official_trial_exclusion`。伪迹标记只用于定义有效数据 segment，不得作为模型输入或 MI 决策条件；该结果不代表真实部署中的在线伪迹检测能力，真实部署仍需要独立的因果在线伪迹检测器。

### 4.1 原生信号存储

模型取数前先构建原生 250 Hz 的干净 segment 信号存储。该存储只负责通道选择、单位转换和伪迹区间物理隔离，不执行滤波、标准化或重采样。

- 每个任务 run 只保留前 22 个 EEG 通道，3 个 EOG 通道不得写入模型信号文件；
- MAT 中的微伏数值乘以 `1e-6` 转为伏特，并保存为通道优先的 `float32` 数组；
- 每个干净 segment 保存为独立文件，文件内容严格对应其原始 run 的 `[start_native, stop_native)`；
- 不得拼接同一 run 中被伪迹区间分开的 segment，也不得重新编号或压缩原始时间坐标；
- 离线窗口和严格在线窗口只保存元数据，使用 `subject/session/run/segment/start/stop` 从同一份 segment 信号中读取，不重复存储重叠窗口；
- 清单必须记录通道顺序、单位、dtype、源 record、原始坐标、相对文件路径和文件哈希，不得写入本机绝对路径；
- 本层输出不包含 128 Hz 兼容数据；兼容重采样必须在后续预处理层按 segment 独立执行并保存两套坐标。

### 4.2 正式 250 Hz 因果滤波

因果窗口级诊断和严格在线仿真共同使用同一份因果滤波 segment 存储；离线窗口级标准使用单独的零相位预处理分支。

- 采用 `scipy.signal.butter(N=4, [8, 30], btype="bandpass", fs=250, output="sos")` 生成 Butterworth 二阶节；这里 `N=4`，带通传递函数的实际总阶数为 8；
- 使用 `scipy.signal.sosfilt` 沿时间轴单向前向滤波，不执行反向滤波、相位补偿或时间戳前移；
- 每个干净 segment 从全零滤波状态独立启动，禁止跨 run、跨 segment 或跨伪迹排除区间继承状态；
- 滤波以 `float64` 计算并保存为通道优先的 `float32` 伏特数组；清单必须保存完整 SOS 系数、实现方式和上游信号清单哈希；
- 每个 segment 开头 250 点（1 秒）定义为滤波启动排除区间；不足 250 点时排除完整 segment；
- 启动区间的滤波输出保留用于审计，但任何与它相交的窗口均不得进入训练、验证、阈值选择或正式测试；
- 过滤后不得压缩或平移原始坐标，正式窗口的 `decision_time_seconds` 仍按原始窗口右端计算，报告延迟时不得扣除群延迟；
- 128 Hz MNE 重采样及零相位滤波不属于本节的正式因果配置；零相位离线分支及其因果性标签按 4.3 和第 5 节执行。

### 4.3 实验分支与因果性声明

第一阶段只运行 250 Hz 实验，但后续数据读取、模型输入和评估记录不得写死 500 点窗口或 125 点步长；128 Hz 兼容配置分别使用 256 点和 64 点，并同时保留原生 250 Hz 坐标和模型坐标。

250 Hz 首轮实验固定为：

| 输出 | 训练预处理 | 验证与测试预处理 | 因果性含义 |
| --- | --- | --- | --- |
| 离线窗口级结果 | 零相位 | 零相位 | 传统非因果离线分类能力 |
| 因果窗口级主诊断 | 因果 | 因果 | 在线模型的单窗诊断 |
| 因果窗口级训练消融诊断 | 零相位 | 因果 | 训练非因果、推理因果 |
| 严格在线仿真主结果 | 因果 | 因果 | 训练与推理均因果 |
| 严格在线仿真训练消融 | 零相位 | 因果 | 训练非因果、在线推理严格因果 |

两种训练方式必须使用相同模型结构、随机种子集合、训练窗口 ID 和训练数量；验证与测试的因果输入、窗口顺序和评价规则必须完全相同。训练消融允许模型分别按同一规则使用因果验证预测选择参数，不能使用最终测试 session。

128 Hz 后续实验尽量复用同一矩阵。当前允许直接使用 MNE 重采样进行兼容；只要重采样会访问未来样本，相应输出就必须命名为“128 Hz 兼容窗口结果”或“128 Hz 兼容在线仿真结果”，不得声明为严格因果。

### 4.4 正式 250 Hz 零相位滤波

离线窗口级结果和零相位训练消融使用按 clean segment 独立构建的零相位信号存储，不得对每个 2 秒窗口分别滤波。

- 使用 MNE 1.11.0 的 `filter_data`，参数固定为 `8–30 Hz`、Hamming 窗、`firwin`、`phase="zero"`、`pad="reflect_limited"`；
- MNE 自动规则在 250 Hz 下得到 413 点 FIR；协议显式冻结滤波长度、2 Hz/7.5 Hz 上下过渡带、全部系数及系数哈希；
- 该对称 FIR 的半支撑长度为 206 点（0.824 秒），内部输出会使用当前样本前后各至多 206 点，因此必须标注为非因果；
- 每个 segment 的前 206 点和后 206 点定义为双侧边缘排除区间；不足 413 点的 segment 没有正式有效样本；
- 边缘输出保留用于审计，但正式窗口必须完整位于 `[start_native+206, stop_native-206)` 内；短 segment 的两侧排除区间允许重叠；
- 禁止先对包含官方伪迹的完整 run 做双向滤波再删除伪迹，以免伪迹通过对称 FIR 污染相邻干净信号；
- 本层不执行标准化或重采样；128 Hz 兼容重采样按 4.3 节另行声明因果性。

### 4.5 Fold 专属标准化

标准化参数使用因果与零相位存储共同合法的离线 Stage 1 训练窗口估计。两种训练分支必须使用逐行相同的窗口 ID；Stage 2 使用其中的任务窗口子集，但与 Stage 1 共用标准化参数。

- 每个 fold 只能使用其 5 个训练 run，验证 run 和测试 session 不得参与均值、标准差或窗口筛选；
- 参数按 EEG 通道独立计算，聚合轴为训练窗口维和时间维；重叠窗口中的同一原始采样每出现一次就计数一次，以匹配模型实际输入和原仓库做法；
- 因果窗口只使用同一批训练 run 的因果统计量，零相位窗口只使用零相位统计量；标准化跟随输入域，而不跟随模型身份；
- 因此零相位训练模型在因果验证、测试和在线推理时，输入必须改用由同一训练 run 估计的因果统计量；
- 标准差不得使用固定下限静默裁剪；出现非有限值或非正值必须立即失败；
- 六折选择完成后，最终模型使用全部 6 个训练 run 的共同窗口重新估计两套参数；
- 清单必须保存训练 run、窗口数量、窗口身份哈希、逐通道参数及全部上游文件哈希；验证和测试只能读取并应用清单，不得重新拟合；
- 标准化后仍以 `float32` 输入模型；窗口长度从预处理清单读取，不得在数据层写死为 500 点。

## 5. 离线窗口级标准

离线标准用于衡量分类器对干净窗口的基本辨别能力，不代表完整在线系统性能。

该分支按 4.4 节使用零相位滤波，并必须明确标注为非因果离线结果。250 Hz 配置优先实现；原仓库的零相位 128 Hz 处理作为后续兼容结果，不阻塞首轮实验。

固定要求：

- 保留原仓库的两阶段任务：Stage 1 为 `IDLE/Task`，Stage 2 为四类 MI；
- 训练和测试必须保持 session 隔离；
- 离线任务窗口以 MI 起点为锚；每个未标记伪迹且保留完整 4 秒 MI 区间的 trial，按 2 秒窗口和 0.5 秒步长生成 5 个重叠任务窗口；
- 离线 `IDLE` 窗口沿用原仓库的 run 起点网格，并且必须完整位于干净 segment 内且不与任何 MI 区间相交；
- 离线窗口与严格在线仿真窗口分表保存，不得用在线 segment 网格替代上述原仓库离线网格；
- 离线窗口使用左闭右开的 `[start, stop)` 区间；`window` 编号在 Stage 1 的每个 run 内从 0 开始，Stage 2 作为其任务窗口子集保留原 Stage 1 编号，不重新编号；
- 任务窗口、纯 `IDLE` 窗口和边界窗口的定义必须写入运行清单；
- 边界窗口可以不计入离线分类指标，但排除规则必须固定且公开；
- Stage 2 在真实任务窗口上的结果必须标注为真值条件诊断结果；
- 最终五分类结果必须包含 Stage 1 的错误传播；
- 每组窗口级结果必须报告实际参与指标计算的窗口总数，以及各真值类别的窗口数量；
- Stage 2 除总体结果外，应按任务窗口相对 MI 起点的 `0/0.5/1.0/1.5/2.0s` 五个固定偏移分别报告诊断指标；这只是对已有预测分组，不移动窗口或补偿滤波延迟；
- 阈值化结果必须注明阈值及其验证集来源。

离线标准至少保留三组输出：

1. Stage 1 的 `IDLE/Task` 结果；
2. Stage 2 在真实 MI 窗口上的四分类诊断结果；
3. 两阶段级联后的最终五分类结果。

具体指标及公式在后续轮次单独冻结。

### 5.1 250 Hz EEGNet OOF 基线

在访问最终测试 session 前，先在训练 session 内执行六折留一完整 run 的 OOF 训练。正式训练器只能读取已冻结的 session0-only bundle；该 bundle 只含训练 session 窗口、fold 统计量及两域训练 segment，训练器不得自动回访联合索引或测试 session 文件。首批矩阵固定为 `6 folds × 2 stages × 2 training domains × 3 seeds = 72` 条 EEGNet 训练轨迹，使用原生 250 Hz、50 epoch、batch size 64、AdamW 和仅由当前 fold 训练标签计算的 balanced cross-entropy。

因果训练轨迹在因果验证输入上记录 logit；零相位训练轨迹在相同验证窗口上同时记录零相位和因果输入 logit，后者用于“零相位训练、因果推理”消融，不增加训练次数。每个 epoch 均保存原始 OOF logit，不在单个 fold 内独立早停。主指标和统一 epoch 选择规则必须在指标轮冻结后只由训练 session 的六折 OOF 结果确定。完整矩阵、恢复和产物规则见 `docs/baseline_training_matrix.md`。

## 6. 严格在线仿真评估标准

严格在线仿真从每个完整原始测试 run 出发，按照冻结的伪迹清单生成全部有效 segment，并逐 segment 消费其中按时间顺序产生的全部窗口，包括所有 MI/IDLE 边界窗口。除已冻结的伪迹及预处理排除规则外，不得依据测试真值再次删除窗口。

固定要求：

- 每个 segment 以首个有效采样点为滑窗网格起点，从可形成完整窗口的位置开始按固定步长推进；
- 构建清单必须记录每个 run 的有效 segment、伪迹排除区间、滤波启动排除区间和因长度不足而无法形成窗口的剩余区间；除这些已记录区间外不得删除数据；
- 所有窗口必须保持原始时间顺序；
- 边界窗口必须进入推理流，不得利用真值提前删除；
- 每个窗口只能使用其决策时间及以前的数据和内部状态；
- 每个窗口都必须计算并保存 Stage 1 和 Stage 2 的原始 logit，不能只保存 `argmax` 标签；
- 两个 Stage 使用同一窗口和同一决策时间；Stage 2 是否参与控制输出由后续决策规则决定；
- 推理时不得读取真实状态、真实类别、未来事件或未来数据；
- 真值只允许在推理完成后用于事件匹配和指标计算；
- 每个 run 和 segment 开始时重置滤波器、模型缓存、投票缓存和离散状态机；
- 每个 run 的有效回放时长及排除区间必须记录；
- 正式 250 Hz 配置不执行重采样；滤波和标准化不得访问未来数据；
- 主结果的训练、验证和测试均使用因果预处理；训练消融允许训练使用零相位信号，但验证、测试和在线推理必须使用与主结果完全相同的因果输入；
- 模型和决策器只能保存协议允许的历史状态；
- 检测延迟必须包含窗口积累造成的时间。

严格在线仿真评估不要求按真实时长等待回放，但在每个决策时刻都不得访问未来信息。在线回放输入使用零相位滤波，或使用会访问未来样本的 MNE 直接重采样至 128 Hz 时，相应结果仅用于调试或兼容对照，不得作为正式严格在线仿真结果；该限制不排除“零相位训练、因果输入在线推理”的训练消融。

模型原始分数记录至少应包含：

```text
subject_id, session_id, run_id, segment_id
window_index
window_start_native, window_stop_native
window_start_model, window_stop_model
decision_time_seconds
stage1_logits, stage2_logits
```

`window_index` 在每个 segment 内从零开始。窗口采用左闭右开区间；`native` 坐标使用原始 250 Hz run 时钟，`model` 坐标使用当前模型输入时钟。正式配置下二者均为 250 Hz；128 Hz 兼容配置必须同时保留两套坐标。`decision_time_seconds` 是相对 run 起点取得完整窗口并允许决策的时刻。Stage 1 和 Stage 2 的 logit 长度分别为 2 和 4。

特定决策策略产生的轨迹应与模型原始分数分开保存，至少包含：

```text
run_id, window_index
decision_state_before, decision_state_after
emitted_class
```

`decision_state_before/after` 分别表示处理当前窗口前后的决策器状态。`emitted_class=-1` 表示没有输出 MI 指令，`1/2/3/4` 表示输出对应 MI 类别。具体决策状态和转换规则在后续轮次确定。

真值时间线应单独保存，不得混入模型输入。

## 7. 离散控制约束

最终系统采用一次 MI 后必须重新经过 `IDLE` 的离散控制范式。

两个 Stage 在所有时刻都持续预测；下列状态只控制能否发出 MI 指令，不控制运行哪个模型。

最小输出状态定义：

```text
READY -> 允许产生一次 MI 触发
WAIT_IDLE -> 已触发 MI，禁止重复输出，等待确认 IDLE
```

固定原则：

- 仅 Stage 1 满足 Task 条件时不改变输出状态，也不会进入等待 Stage 2 的中间状态；
- 在 `READY` 中，Stage 1 的 Task 证据与 Stage 2 的类别证据联合满足条件时，才输出一次 MI；
- 两组证据可以分别聚合截至当前时刻的历史分数，但不得使用未来窗口；
- 输出 MI 后立即进入 `WAIT_IDLE`，两个 Stage 继续预测，但新的 MI 预测不得产生控制输出；
- 系统不会在输出 MI 后强制假定已经进入 `IDLE`；只有 Stage 1 的因果 `IDLE` 证据满足确认条件后，才返回 `READY`；
- 未能确认 `IDLE` 应记录为复位失败，不得静默强制复位；
- 状态转换参数必须由训练 session 的验证数据确定；
- 无状态结果与加入离散先验的结果必须分开报告。

具体分数聚合方式、置信度阈值、连续确认窗口数和事件匹配容差在后续轮次确定，并与检测延迟共同报告。

## 8. 复现与审计要求

每次正式评估必须保存运行清单，至少记录：

- Git commit；
- 数据集、被试、session 和 run；
- 冻结数据文件、构建清单版本及文件哈希；
- 预处理、窗口和步长参数；
- 数据划分及验证参数来源；
- checkpoint 路径及文件哈希；
- 阈值、状态机和事件匹配配置；
- 随机种子与运行环境；
- 预测文件、指标文件和日志路径。

任何协议字段缺失的结果只能用于调试，不得作为正式可比较结果。

## 9. 后续待冻结项目
